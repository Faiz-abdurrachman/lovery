# 2. Filosofi Design

Lovery percaya bahwa proses memesan jasa fotografi seharusnya terasa sederhana, nyaman, dan menyenangkan.

Website tidak dibuat untuk terlihat rumit.

Website dibuat agar calon klien:

- mudah memahami layanan
- mudah menghubungi admin
- mudah mengajukan sesi
- percaya terhadap kualitas Lovery

Sedangkan Dashboard dibuat untuk:

- mempercepat pekerjaan admin
- mengurangi pekerjaan manual
- meminimalkan kesalahan administrasi

# 32. Do & Don't

## ✅ Do

- Gunakan whitespace
- Gunakan foto sebagai fokus utama
- Gunakan Bahasa Indonesia
- Gunakan Design Token
- Gunakan komponen reusable

## ❌ Don't

- Jangan menggunakan istilah "Booking"
- Jangan membuat dashboard penuh warna pink
- Jangan membuat gradient berlebihan
- Jangan menggunakan lebih dari 2 font
- Jangan membuat komponen baru tanpa alasan
- Jangan menggunakan warna di luar Design System

# Ringkasan

Design System menjadi pedoman utama seluruh desain Lovery Studio Management System.

Dokumen ini memastikan seluruh desain memiliki identitas visual yang konsisten, pengalaman pengguna yang nyaman, serta implementasi frontend yang mudah dan seragam.

Semua perubahan pada desain harus mengacu pada dokumen ini dan disetujui oleh Product Team sebelum diterapkan.
